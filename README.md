# ⚔️ 내전 고?

LoL 내전(사용자 설정 게임) 팀 밸런싱 & 전적 분석 웹앱.
티어·포지션·챔프 폭·자체 MMR을 종합해 가장 균형 잡힌 5:5 조합을 찾고, 결과를 축적해 분석합니다.

## 실행 방법

1. **`start-dev.bat` 더블클릭** (최초 1회는 의존성 자동 설치)
2. 브라우저에서 **http://localhost:3000** 접속
3. 종료: 까만 창에서 `Ctrl + C`

> 수동 실행: `npm install` 후 `npm run dev`
> (Node.js는 `C:\Program Files\nodejs`에 설치되어 있습니다)

## 기능

| 페이지 | 설명 |
|--------|------|
| **내전 만들기** | 10명 선택 → 밸런스 자동 생성 → 차선 조합 비교 → 내전 저장 |
| **선수 관리** | 선수 등록/수정/삭제, 주 포지션 순위, 챔프 폭, 수동 보정 |
| **히스토리** | 지난 내전 목록, 결과 입력(승팀+챔프) → MMR·전적 자동 갱신 |
| **분석** | 밸런스 정확도, 개인 전적·MMR 랭킹, 챔피언별 승률 |

## 밸런싱 알고리즘

- 10명을 두 팀으로 나누는 **126가지 조합을 완전 탐색**
- 각 팀에 **5포지션을 겹침 없이 최적 배정**(주 포지션 최대 반영)
- **티어 점수(비선형, 상위 가중) + 자체 MMR + 수동 보정**으로 실력 점수 산출
- 전력 균형을 우선하되, 비슷한 조합 중 포지션 만족도가 높은 쪽 선택

## 자체 MMR

- 내전 결과로 갱신되는 Elo 레이팅 (초기값 = 티어 점수)
- 약팀이 이기면 많이 오르고, 강팀이 지면 많이 내림

## 기술 스택

- **Next.js (App Router) + TypeScript + Tailwind CSS v4**
- **Supabase** (PostgreSQL) — 선수/매치 데이터 저장
- 환경변수는 `.env.local` (Git 미포함)

## 프로젝트 구조

```
app/            페이지 (내전 만들기/선수/히스토리/분석)
components/      UI 컴포넌트
lib/
  types.ts      타입 정의
  tier.ts       티어 → 점수 변환 (비선형)
  balance.ts    밸런싱 알고리즘
  elo.ts        자체 MMR(Elo) 갱신
  stats.ts      통계 계산
  db.ts         Supabase CRUD
  supabase.ts   Supabase 클라이언트
supabase/
  schema.sql    DB 테이블 스키마
```
