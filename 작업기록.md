# 📋 내전 고? — 작업 인수인계 문서

> 다른 컴퓨터/새 Claude 세션에서 이 프로젝트를 이어서 작업할 때 이 문서를 먼저 읽으면 전체 맥락을 파악할 수 있습니다.

## 프로젝트 개요
- **무엇:** LoL 내전(사용자 설정 게임, 10명 5:5) 팀 밸런싱 & 전적 분석 웹앱
- **이름:** 내전 고? (naejeongo)
- **목표:** 티어·포지션·챔프폭·자체 MMR을 종합해 균형 잡힌 팀을 자동 구성하고, 결과를 축적해 분석

## 라이브 & 저장소
- **배포 URL:** https://naejeongo.vercel.app
- **GitHub:** https://github.com/inhye-source/naejeongo
- **배포 방식:** Vercel CLI (Vercel 팀 스코프: `inhye`, 프로젝트명: `naejeongo`)

## 기술 스택
- Next.js 15 (App Router) + TypeScript + Tailwind CSS v4
- Supabase (PostgreSQL) — 선수/매치 데이터 저장
- Anthropic Claude API (`claude-opus-4-8`) — 스크린샷 소환사명 인식(비전)
- html-to-image — 결과 이미지 캡처

## 실행 방법
1. `npm install`
2. `.env.local` 생성 (아래 환경변수)
3. `npm run dev` → http://localhost:3000
   - (Windows에서 `npm` 실행 차단 시: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 후 Y, 또는 `npm.cmd` 사용)

## 환경변수 (.env.local — GitHub에 없음, 수동 생성 필요)
```
NEXT_PUBLIC_SUPABASE_URL=https://hrsgcisvsctkvtvtjjce.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=(Supabase anon key)
```

> ⚠️ Vercel CLI로 env 등록 시 PowerShell이 값 끝에 `\r`을 붙여 깨지는 이슈가 있었음 → **Vercel REST API로 등록**해야 안전 (과거에 한 번 해결함).

## 구현된 기능
| 페이지 | 기능 |
|--------|------|
| 내전 만들기 (`/`) | 10명 선택 → 밸런스 생성 → 차선 조합 비교 → 저장. 스크린샷 자동구성, 같은팀 고정그룹, 티어 점수표, 결과 공유(텍스트/이미지), 부등호 우세표시, 팀 색상 |
| 선수 관리 (`/players`) | 등록/수정/삭제, 주포지션 순위, 수동 보정, 샘플 14명 |
| 히스토리 (`/history`) | 지난 내전 목록, 결과 입력(승팀+챔프) → MMR·전적 자동 갱신 |
| 분석 (`/stats`) | 밸런스 정확도, 개인 전적·MMR 랭킹, 챔피언별 승률 |

## 밸런싱 알고리즘 (lib/balance.ts)
- 10명을 두 팀으로 나누는 126가지 조합 완전 탐색
- 각 팀 5포지션 최적 배정 (주포지션 최대 반영)
- 실력점수 = 티어점수(비선형, 상위가중) + 자체 MMR + 수동보정
- 고정 그룹(lockGroups): 지정 인원이 한 팀에 온전히 들어가는 조합만 탐색
- 비용 = 전력차 + 포지션불만족×60 (전력 균형 우선)

## 티어 점수 (lib/tier.ts) — 비선형(상위 가중)
챌린저 4600 / GM 3600 / 마스터 2800 / 다이아 2000~2600 / 에메랄드 1500~1800 / 플래 1100~1400 / 골드 750~975 / 실버 450~675 / 브론즈 200~350 / 아이언 0~150

## 자체 MMR (lib/elo.ts)
- Elo 방식, 초기값 = 티어점수, K=80
- 약팀이 이기면 많이 오르고 강팀이 지면 많이 내림

## 결정 사항 (인터뷰 결과)
